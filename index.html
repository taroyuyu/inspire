Introducing spires.app ...
Watch the instruction video!
What does it do?
  • It caches SPIRES queries for faster lookups.
  • It grabs abstracts from arXiv.org or journal websites.
  • Download & read PDFs by one click.
  • You can even read your favorite arXiv/new.
Setting up
Please download the app. Mac OS X Leopard is required. Needs at least 10.5.7! Please update it if you haven't.
Then, please open the preferences pane by spires→Preferences, and set the directory for keeping PDFs.
If you don't, ~/Desktop is used by default. It will soon clutter your desktop into a mess, though.
If you already have PDFs named as 0808.0808.pdf or hep-th0123456.pdf, please move them into the directory you specified, or make symbolic links to them in it.
Basic Usage
  1. Type a spires query to the search box. e.g.
    t monopole
    or
    a Weinberg
    Here, 't' stands for the 'title', and 'a' stands for the 'author'.
  2. Hit the enter key.
  3. arXiv/new can be refreshed by a right click on it.
Query Syntax
  • Currently only supports
    • a(uthor) name,
    • c(ites) key
    • d(ate) year
    • ea name.
    • e(print) number,
    • j(ournal) journal.
    • t(itle) title,
    • to(pcite) number,
    and their simple "and" combinations.
  • Example: a Maldacena and t holography and d > 2000
System Hook
By going spires→Install System Hook, you can add "search in spires" menu to the contextual menus of a few apps.
The hook is installed in /Library/InputManagers/.
Known to works with

Known Problems
Many.
  • It slows down to a crawl once there are more than 50000 entries. ArXiv has ~ 5 x 105 entries, so I'd like to make spires.app easily handle that amount, but it seems to need a significant change in plumbing.
If you have any specific bug reports or comments, please contact the author via spires→Send Bug Reports.
Source code
  • Source code can be viewed online at the bzr repository. (thanks to Jacques Distler for hosting)
  • To download the source,
    1. Get bzr at Bazaar website, install it, and read the tutorial.
    2. Create a folder to keep the files, and do
      bzr branch http://www.sns.ias.edu/~yujitach/spires/code/
    3. To update to the latest source code, do
      bzr merge
  • To build,
    1. Double-click BWToolKit.ibplugin once to register it to the Interface Builder
    2. Open spires.xcodeproj and build the project.
